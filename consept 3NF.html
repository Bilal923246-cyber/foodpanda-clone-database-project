🎓 E.F. Codd's Rule · Third Normal Form (3NF)

📘 Third Normal Form (3NF) | Eliminating Transitive Dependency

🔍 Complete visual guide: Rules, transitive dependency definition, examples, candidate key analysis, and formal 3NF test condition
📌 Two Essential Rules for 3NF

To bring a table into Third Normal Form (3NF), it must satisfy:

  • Rule 1: The table must already be in Second Normal Form (2NF) – no partial dependencies.
  • Rule 2: There must be no transitive dependency – meaning no non-prime attribute should be functionally dependent on another non-prime attribute.
💡 Why 3NF matters: It removes transitive dependencies, further reducing data redundancy and eliminating update/insertion/deletion anomalies that occur when non-key attributes depend on other non-key attributes.
🧠 What is Transitive Dependency?

A transitive dependency occurs when a non-prime attribute determines another non-prime attribute through a chain of dependencies.

Formally: For attributes X, Y, Z, if X → Y and Y → Z, then X → Z transitively. If Y is a non-prime attribute and Z is also non-prime, this is a transitive dependency violation for 3NF.

⚠️ Critical Rule: A non-prime attribute must not determine another non-prime attribute.
✅ Allowed cases: Prime → Prime, Prime → Non-prime, Non-prime → Prime.
❌ Not allowed: Non-prime → Non-prime.
📊 Example 1: Student Table (Violates 3NF due to Transitive Dependency)

Consider a Student table with attributes: Roll_Number, State, City and functional dependencies:

  • Roll_Number → State
  • State → City

Step 1 – Candidate Key: Roll_Number is the candidate key (and primary key).
Step 2 – Prime vs Non-Prime:
✅ Prime attribute: Roll_Number
❌ Non-prime attributes: State, City
Step 3 – Check transitive dependency:
Roll_Number → State (Prime → Non-prime) → ✅ allowed.
State → City (Non-prime → Non-prime) → ❌ Transitive dependency violation!
Because Roll_Number → City transitively through a non-prime attribute State.

❌ Violation: This table is NOT in 3NF because a non-prime attribute (State) determines another non-prime attribute (City).

Solution: Decompose into two tables:
Student_Info: (Roll_Number, State)
State_City: (State, City)

📐 Formal 3NF Condition (Question Solving Approach)

For a relation to be in Third Normal Form (3NF), for every functional dependency X → Y (where X and Y are sets of attributes), at least one of the following must hold:

✅ Condition 1: X is a Superkey (or Candidate Key).

✅ Condition 2: Y is a Prime Attribute (part of any candidate key).

In other words: Either the left-hand side is a superkey, OR the right-hand side is a prime attribute.

🎓 Exam Tip: This is the standard definition used in GATE, UGC NET, and university exams. You must find all candidate keys first, then check every FD against this rule.
🔍 Example 2: R(A, B, C, D) with FDs AB → CD and D → A

Given: Relation R(A, B, C, D)
Functional Dependencies:
1. AB → CD
2. D → A

Step 1 – Find all Candidate Keys:
- Compute closure of AB: AB → AB (reflexive), AB → CD (given) → ABCD → AB is a candidate key.
- Since D → A, we can replace A with D. Check closure of DB: DB → DB (reflexive), D → A gives A, then AB → CD gives C and D → DB is also a candidate key.
✅ Candidate Keys: {AB, DB}

Step 2 – Identify Prime and Non-Prime Attributes:
Prime attributes: A, B, D (all participate in candidate keys)
Non-prime attribute: C (only C is not part of any candidate key)

Step 3 – Check each FD against 3NF condition:
FD1: AB → CD
Left-hand side AB is a candidate key → ✅ satisfies Condition 1 (LHS is superkey).
FD2: D → A
Left-hand side D is not a candidate key (D alone is not superkey). But check Condition 2: Right-hand side A is a prime attribute (since A is part of candidate keys). → ✅ satisfies Condition 2.

✅ Conclusion: Both FDs satisfy the 3NF condition. Therefore, R is in Third Normal Form (3NF).
📚 Example 3: R(A, B, C) with FDs A → B, B → C

Given: Relation R(A, B, C) with FDs: A → B, B → C
Candidate Key: A (closure A → A,B,C)
Prime attributes: A   Non-prime: B, C
Check FD A → B: LHS A is candidate key → ✅ OK.
Check FD B → C: LHS B is not a candidate key. RHS C is non-prime → ❌ violates 3NF (transitive dependency: A → B → C).
Result: Not in 3NF. Decompose into R1(A, B) and R2(B, C).

📐 Step-by-Step Algorithm to Test for 3NF

Follow these steps to determine if a given relation is in Third Normal Form:

  1. Step 1: Ensure the relation is already in 2NF (no partial dependencies).
  2. Step 2: Find all candidate keys of the relation using attribute closure method.
  3. Step 3: Identify prime attributes (attributes that belong to any candidate key) and non-prime attributes.
  4. Step 4: For every given functional dependency X → Y, check:
    - Is X a superkey (candidate key or contains a candidate key)?
    - OR is every attribute in Y a prime attribute?
    If both conditions fail for any FD → the relation is NOT in 3NF.
    Otherwise, it is in 3NF.
🎯 Key Insight: Unlike 2NF (which only cares about composite keys and partial dependencies), 3NF handles transitive dependencies and can be checked using the superkey/prime condition.
📋 Quick Reference: 1NF → 2NF → 3NF
Normal FormRequirement
First Normal Form (1NF)Atomic values, no repeating groups / multivalued attributes.
Second Normal Form (2NF)1NF + No partial dependency (non-prime attributes fully depend on whole candidate key).
Third Normal Form (3NF)2NF + No transitive dependency (non-prime attribute → non-prime attribute is forbidden).
Equivalent: For every FD X→Y, either X is superkey OR Y is prime attribute.

Memory Aid:
- 1NF: Atomic cells.
- 2NF: Whole key (no partial dependency).
- 3NF: Nothing but the key (no transitive dependency).

💡 Important Clarification: What is not a violation?

Many students get confused about which dependencies are allowed in 3NF. Remember:

  • ✅ Prime → Prime (e.g., Candidate key attribute determines another prime attribute) – Allowed.
  • ✅ Prime → Non-prime (e.g., Candidate key determines a non-key attribute) – Allowed.
  • ✅ Non-prime → Prime (e.g., A non-key attribute determines a key attribute) – Allowed (this is rare but valid).
  • ❌ Non-prime → Non-prime – NOT allowed (this is the transitive dependency violation).
🔑 The only forbidden pattern in 3NF: A non-prime attribute determining another non-prime attribute.
